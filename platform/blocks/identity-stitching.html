How identity stitching works

Real-time graph resolution

As signals arrive, Blocks matches identifiers, merges profiles, and back-fills history across all linked IDs — and guardrails make sure the graph never merges two different fans into one.

  1. 01 {% include icon.html name="link" class="h-5 w-5" %}

    Real-time graph building

    As events arrive, identifiers are matched and profiles merged in real time — the fan graph grows with every new signal, not overnight in a batch job.

  2. 02 {% include icon.html name="users" class="h-5 w-5" %}

    Multi-identifier matching

    Match on email, device ID, app ID, ticket number, loyalty ID, custom identifiers, or any combination — instead of forcing identity onto one hard key that is often missing.

  3. 03 {% include icon.html name="repeat" class="h-5 w-5" %}

    Automatic history back-fill

    When profiles merge, new resolutions back-fill history across all linked identifiers — the complete relationship reads as one story from first touch to present.

  4. 04 {% include icon.html name="shield" class="h-5 w-5" %}

    Guardrails against false merges

    Per-identifier limits stop a shared device from over-merging distinct fans. When a limit would be crossed, records hold apart instead of guessing — correct beats complete-but-wrong.

Under the hood

Match, merge, back-fill

  1. 01

    Match on evidence, not hope

    Every event carries the identifiers its source can see. When a new event shares an identifier with an existing thread — a login, a loyalty scan, a clicked email — that is evidence, and evidence is the only thing that ever joins two records.

    →
  2. 02

    Merge in the moment

    The matched profiles fold into one golden record immediately — identifiers, attributes, and consent state combined — so the very next event anywhere in your stack already sees the full fan.

    →
  3. 03

    Back-fill the past

    Everything the separate records saw before the merge is rewritten onto the fan: Monday's anonymous session becomes their session. And when a shared device would cross a per-identifier limit, the records hold apart — because a smaller true graph beats a bigger wrong one.

    →
